# OBI Protocol

When the user types `/obi`, `@obi`, or asks to sync project documentation to
Obsidian, follow `~/.obi/protocol/OBI_PROTOCOL.md`.

Use Obsidian MCP if available; otherwise write directly to the configured vault
path in `~/.obi/config.toml`.

Do not create duplicate docs with names like `final`, `new`, or date suffixes.
Update canonical active docs in place and write `Sync Report.md`.
